# SDC 고객품질 분석 AI 시스템 - 설명회 시나리오

## 시스템 개요

**SDC 고객품질 분석 AI 시스템**은 디스플레이 제품의 품질 불량 분석을 AI 기반으로 자동화하는 시스템입니다.

- **프론트엔드 (품질분석 Chatbot)**: http://localhost:3000
- **결함분석시스템 (Gradio)**: http://localhost:7860

---

## 설명회 시나리오

### 1단계: 시스템 소개 (2분)

**[화면: http://localhost:3000 메인 화면]**

> "안녕하세요. SDC 고객품질 분석 AI 시스템을 소개드리겠습니다.
>
> 이 시스템은 고객사로부터 접수된 품질 불량 건에 대해 AI를 활용하여
> 데이터 수집, 분석, 보고서 작성까지 전 과정을 지원합니다."

**주요 기능 카드 설명:**
1. **고객불량 확인** - 고객사 불량 메일 수신 및 접수
2. **데이터 수집** - MES, 품질검사, 공정/설비 이력 조회
3. **기본 분석** - 1차 분석, 담당자 정보, 귀책부서 분석
4. **상세 분석** - 현장 데이터, 집계 데이터 심층 분석
5. **보고서 작성** - 과거 보고서 참조, 분석서 작성, 내부 결재
6. **과거 이력확인** - 유사 사례 조회 및 연계

---

### 2단계: 고객불량 메일 접수 (3분)

**[클릭: "고객불량 분석요청 메일 수신/접수" 버튼]**

> "고객사에서 불량 관련 영문 메일이 도착했습니다.
> 시스템이 자동으로 메일을 파싱하여 주요 정보를 추출합니다."

**시연 포인트:**
1. 영문 메일 원문 표시
2. **[번역 버튼 클릭]** → AI가 자동 한글 번역
3. 추출된 정보 확인:
   - 고객사: Apple
   - 제품 모델: OLED_67_FHD
   - LOT ID / Cell ID
   - 결함 유형: DEAD_PIXEL
4. **[접수 시스템으로 연계]** 버튼 클릭 → CS 워크플로우 연동
5. **히스토리 자동 저장** → "고객불량 확인" 그룹에 이력 생성

---

### 3단계: 데이터 수집 (4분)

**[클릭: "데이터 수집" 카드]**

> "접수된 불량 건에 대해 관련 데이터를 수집합니다.
> 5가지 카테고리의 데이터를 조회할 수 있습니다."

**시연 포인트:**
1. **MES 생산 실적** 조회
   - 생산 수량, 수율 정보
   - 공정 흐름 추적 (Array → Cell → Module)

2. **품질검사 이력** 조회
   - 검사 항목별 결과
   - 불량 이력 및 판정 정보

3. **공정/설비 이력** 조회
   - 설비 파라미터
   - 이상 발생 이력

4. **[기본 분석 시작]** 버튼 클릭
   - 수집된 데이터가 컨텍스트로 AI에 전달
   - "데이터수집" 그룹에 이력 자동 저장

---

### 4단계: AI 기반 분석 (3분)

**[채팅 인터페이스에서 분석 진행]**

> "수집된 데이터를 기반으로 AI가 분석을 수행합니다."

**시연 포인트:**
1. **1차 분석** 요청
   - AI가 데이터 패턴 분석
   - 잠재적 원인 도출

2. **귀책부서 분석** 요청
   - 공정별 책임 분석
   - 근본 원인 추정

3. **우측 사이드바** 활용
   - 분석 단계별 액션 버튼
   - 고객 담당자 연락처 조회
   - 이메일 초안 작성 지원

---

### 5단계: 유사 사례 조회 (2분)

**[클릭: "유사 사례 조회" 버튼]**

> "과거 유사한 불량 사례를 조회하여 분석에 참고합니다."

**시연 포인트:**
1. 현재 분석 대상과 유사한 과거 사례 목록 표시
2. 유사도 점수 및 주요 정보 확인
3. **[분석에 활용]** 클릭 → 과거 사례 컨텍스트가 AI에 전달
4. AI가 과거 사례를 참고하여 분석 정확도 향상

---

### 6단계: 과거 보고서 조회 (2분)

**[클릭: "과거 보고서 수집" 버튼]**

> "과거 분석 보고서를 조회하여 보고서 작성에 참고합니다."

**시연 포인트:**
1. 과거 보고서 목록 팝업
2. 검색 및 필터링 기능
3. 보고서 상세 보기
   - 고객사, 제품, 결함 유형
   - 원인 분석 결과
   - 대응 조치 내용
4. 복사/인쇄 기능

---

### 7단계: 채팅 히스토리 관리 (2분)

**[좌측 히스토리 사이드바]**

> "모든 분석 이력은 자동으로 저장되어 관리됩니다."

**시연 포인트:**
1. **그룹 구조**
   - 고객불량 확인
   - 데이터수집
   - 기본분석 저장

2. **더보기 메뉴 (...)** 클릭
   - **수정**: 채팅 제목 변경
   - **복제**: 채팅 내용 복사
   - **메일전송**: 채팅 내용을 이메일로 전송
   - **삭제**: 채팅 삭제

3. **우클릭 컨텍스트 메뉴**
   - 그룹 간 이동 기능

---

### 8단계: 결함분석시스템 연동 (2분)

**[클릭: 헤더의 "결함분석시스템" 버튼]**

> "더 상세한 결함 분석이 필요한 경우, 결함분석시스템으로 이동할 수 있습니다."

**시연 포인트:**
1. Gradio 기반 결함분석시스템 (http://localhost:7860)
2. CS 워크플로우 전체 프로세스
3. VLM 기반 이미지 분석
4. GraphRAG 기반 지식 그래프
5. **[고객품질분석시스템]** 버튼으로 다시 복귀

## 주요 특징 요약

| 기능 | 설명 |
|------|------|
| AI 번역 | 영문 메일 자동 한글 번역 |

## Q&A 예상 질문

**Q1. AI 모델은 어떤 것을 사용하나요?**
> Ollama 기반 로컬 LLM을 사용합니다. 보안상 외부 API 없이 온프레미스에서 운영됩니다.

**Q2. 데이터는 실제 시스템과 연동되나요?**
> 현재는 목업 데이터로 시연하며, 실제 MES/품질시스템과 API 연동 가능합니다.

**Q3. 분석 결과의 정확도는?**
> AI 분석은 참고 자료로 활용하며, 최종 판단은 품질 담당자가 수행합니다.

**Q4. 오프라인 환경에서 작동하나요?**
> 네, 완전 오프라인 환경에서 작동합니다. 외부 네트워크 연결이 필요 없습니다.
